# v0.28 — Konsumer-Patterns Backlog

**Status:** Sammlung · 2026-05-06 (anlässlich v0.25.6-Diskussion)
**Zweck:** Sammelpunkt für alles, was wegen Single-File-Anker (ADR-PC5) NICHT im Standard-Inspector gelandet ist, aber als **separate Konsumer-Variante** ab v0.28 Sinn ergibt.

> v0.28 wird Schaufenster für **"Inspector Pro"-Click-Through-Pattern**: Standard-Inspector bleibt schlank/Single-File, Pro-Variante bündelt Polyfills/3D/Composition für Power-User.

---

## 1. Architektur-Pattern (vorgeschlagen 2026-05-06)

### ADR-PC6-Skizze: Click-Through statt Bundle-Bloat

**Kontext:** Mehrfach im 2026-05-Sprint-Zyklus (v0.25.2 QR-Pivot, v0.25.5 Texture-Polyfill-Verzicht, v0.25.7 USDC-Material-Limitation) wurden Features verworfen, weil sie Single-File-Anker oder Privacy-First brechen. Lösung war jeweils "ehrlich rauslassen + Hint im UI" — aber das frustriert Power-User.

**Vorschlag:** v0.28 etabliert **Click-Through-Pattern**:

```
[Inspector Standard v0.27]            [Inspector Pro v0.28+]
  ├─ Single-File HTML, schlank          ├─ Mit Polyfills (KTX2/TIFF/ASTC)
  ├─ Privacy-First                      ├─ Mit Three.js Desktop-3D
  ├─ Lokal-öffenbar                     ├─ Mit usd-wasm Composition
  ├─ "Mehr Features?  →"                ├─ Mit MCP-LLM-Findings-Erklärung
  └─ Click führt zu Pro-URL             └─ Eigene URL, eigenes Hosting
```

**User wählt per Click**, was er für seinen Use-Case braucht. Beide Varianten Privacy-First (USDZ verlässt nie den Browser).

**Konsequenz:** Single-File-Anker bleibt für Standard-Inspector. Pro-Variante ist parallele zweite Distribution mit eigenem Hosting (z. B. `kopfkinok3.github.io/usdseal-inspector-pro/`). Beim Click-Through muss User USDZ neu droppen (Cross-Domain-Privacy — kein Magic-Transfer möglich, das ist OK und ehrlich).

```
v0.28.4  Inspector Pro: USDC-Composition     (usd-wasm-Embed) — wenn nötig

```

Jeder Sub-Release ist **eigenes HTML + eigenes Hosting**, NICHT im Standard-Inspector-Repo gepflegt. Standard-Inspector hat nur den Click-Through-Link.

---

## 4. Strategischer Hebel

1. **Glaubwürdigkeit:** Standard-Inspector bleibt Privacy-First-/Single-File-Anker treu — "wir verkaufen kein Sandwich-Bundle".
2. **Power-User-Pfad existiert:** Wer mehr braucht, klickt sich zur Pro-Variante. Kein "wir können das nicht" mehr.
3. **PR-Story:** "Inspector hat zwei Gesichter: schlank für 90 % der User, Pro für die anderen 10 % — Du wählst."
4. **AOUSD-Talk:** Click-Through-Demo zeigt beide Welten. Der Standard-Inspector ist die Story, Pro ist der Bonus-Slide.
